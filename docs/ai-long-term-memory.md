# AI 长期记忆说明

## 目标
- 在本地优先前提下，为 AI 提供跨会话可复用的长期偏好与事实。
- 采用“半自动确认”：AI 只提取候选，用户确认后才入库。

## 什么该记录
- 稳定偏好：输出格式、语言、风格、结构偏好。
- 长期事实：岗位、常用工具、固定工作上下文。
- 持续目标：长期项目目标、里程碑、约束条件。

## 什么不该记录
- 敏感信息：密钥、密码、证件号、银行卡、手机号、邮箱等。
- 一次性信息：验证码、临时链接、短时事务、会话性数据。
- 未确认推断：模型猜测但用户未确认的内容。

## 同步与会员规则
- `active` / `expiring_soon`：允许 `ai_memory` 云端 push/pull。
- `expired`：`ai_memory` 仅本地可读写，云端同步停用。
- 到期前 3 天仅提醒，不改变同步行为；到期即停。

## 开关含义
- `enable_long_term_memory`：长期记忆总开关。
- `enable_memory_auto_recall`：发送消息前自动召回相关记忆。
- `enable_memory_auto_save`：从“记住类”输入自动提取候选。
- `enable_memory_sync`：长期记忆参与个人云同步。
